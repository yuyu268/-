課題５
歌手とそのアルバムを管理するという点で、現状のAPIの改善点を検討し思いつく限り書き出してください。

・データ量が増えた際に目的の歌手やアルバムを見つけやすくなるよう、IDだけでなく歌手やアルバムの名前でも該当の歌手やアルバムが表示できるようにする
・現時点ではサーバーが何らかの原因で落ちたりなど、稼働を終了してしまった際に追加や削除などの変更もリセットされてしまうため、そのようなことがあっても立ち上げ直した際にデータが残るようにする
・歌詞やアルバムを追加した際に、全く同じデータを二重に追加してしまった際に「すでに同じデータが追加されています」などのメッセージを表示する
・書き込もうとしたIDが既に使用されていた際に警告のようなメッセージを表示する
・現在はIDの指定が必須であり、データ量が増えた際にどのIDを使用しているか管理が困難になる可能性があるため、IDを指定していない状態であれば、未使用の最小値のIDで自動で割り振り追加する機能の実装
・複数のデータを一括で削除、追加する機能の実装
・歌手の取得でもアルバムの情報を付加する
・APIであるため必要はないかもしれないが、横並びだと見づらいため、キーごとに改行し縦並びに表示するようにする
